OpenAI announced Thursday that it has entered into an agreement to acquire Astral, the company behind popular open source Python development tools such as [uv](https://github.com/astral-sh/uv), [Ruff](https://github.com/astral-sh/ruff), and [ty](https://github.com/astral-sh/ty), and integrate the company into its Codex team.

The deal, whose financial terms were not publicly disclosed, will help OpenAI “accelerate our work on Codex and expand what AI can do across the software development lifecycle,” the company [said in an announcement post](https://openai.com/index/openai-to-acquire-astral/). Integrating Astral’s tools more closely with Codex after the acquisition will “enable AI agents to work more directly with the tools developers already rely on every day,” it continued.

Astral’s most popular open source projects include:

-   uv, a Rust-based Python package manager that helps manage the complex web of dependencies in Python coding environments ([attracted over 126 million monthly downloads](https://pypistats.org/packages/uv))
-   Ruff: A Python [linter](https://www.testim.io/blog/what-is-a-linter-heres-a-definition-and-quick-start-guide/) and code formatter ([179 million monthly downloads](https://pypistats.org/packages/ruff))
-   Ty: A fast Python type-checker, currently in beta ([19 million monthly downloads](https://pypistats.org/packages/ty))

Charlie Marsh, who [founded Astral three years ago](https://astral.sh/blog/announcing-astral-the-company-behind-ruff) with $4 million in seed funding, promised in [a blog post Thursday](https://astral.sh/blog/openai) that OpenAI “will continue supporting our open source tools after the deal closes. We’ll keep building in the open, alongside our community – and for the broader Python ecosystem – just as we have from the start.” OpenAI echoed that, following the close of the acquisition, saying it will “continue to support these open source projects while exploring ways they can work more seamlessly with Codex.”

OpenAI’s move comes amid an [intense battle](https://arstechnica.com/ai/2026/02/openai-picks-up-pace-against-claude-code-with-new-codex-desktop-app/) between [Codex](https://arstechnica.com/ai/2026/02/with-gpt-5-3-codex-openai-pitches-codex-for-more-than-just-writing-code/) and [Anthropic’s Claude Code](https://arstechnica.com/tag/claude-code/) for dominance in the fast-growing market for AI-powered coding assistants. Back in November, [Anthropic acquired Bun](https://www.anthropic.com/news/anthropic-acquires-bun-as-claude-code-reaches-usd1b-milestone), a JavaScript runtime with 7 million monthly downloads that the company said would lead to “faster performance, improved stability, and new capabilities” after its integration into Claude Code.

Earlier this month, [OpenAI acquired Promptfoo](https://openai.com/index/openai-to-acquire-promptfoo/), the makers of an open source security tool focused on LLMs.
